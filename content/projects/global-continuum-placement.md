# Global Continuum Placement

Global Continuum Placement addresses workload placement across distributed computing resources in the PHYSICS project context. It is relevant to Cloud-HPC work because placement decisions must account for heterogeneous compute sites, application requirements, and infrastructure constraints.

## Contribution

- Contributed to Python services and placement logic for distributed workload scheduling.
- Worked with FastAPI, aiohttp, Docker, and Poetry for reproducible development.
- Helped connect scheduling concepts with workflow and infrastructure execution concerns.
- Supported evaluation and documentation around meta-scheduling behavior.

## Technical Context

- **Scheduling focus**: Workload placement and meta-scheduling.
- **Backend**: Python, FastAPI, aiohttp.
- **Infrastructure**: Docker-based reproducibility and distributed resource models.
- **Research context**: PHYSICS EU research project.

## Relevance

This project is a bridge between research and platform engineering: optimization and scheduling ideas are implemented as software that can interact with real workflow and infrastructure systems.
